# **Fling**

![Tests](https://img.shields.io/badge/tests-231%20passing-brightgreen)
![Coverage](https://img.shields.io/badge/coverage-100%25-brightgreen)
![License](https://img.shields.io/badge/license-MIT-blue)
![Node](https://img.shields.io/badge/node-%3E%3D18-green)

Fling is a statically typed, recipe-themed programming language where writing code feels like cooking something up from scratch. Programs are made of ingredients, steps, recipes, prep types, simmering loops, and final serves, because programming and cooking share the same DNA: gather inputs, follow a process, and yield something delicious. Its killer feature is a compile-time unit type system, so values like `200g`, `350ml`, `2cups`, and `35minutes` carry meaning instead of floating around as mystery numbers. Add grams to grams and Fling is happy; add grams to milliliters and the compiler sends it back to the kitchen. 🍳

## Quick Example

**Fling Source**

```fling
ingredient greeting: label = "Hello from Fling!"
serve greeting
```

**Generated JavaScript**

```javascript
// Generated by the Fling compiler
// Do not edit by hand
function display(x) { console.log(x); }
function size(col) { return col.length; }
const __unitFactors = { grams: 1, kg: 1000, oz: 28.35, lbs: 453.59, ml: 1, liters: 1000, cups: 236.6, tbsp: 14.79, tsp: 4.93, floz: 29.57, celsius: 1, seconds: 1, minutes: 60, hours: 3600 };
function convert(val, unit) {
  if (unit === "fahrenheit") return val * 9 / 5 + 32;
  return val / __unitFactors[unit];
}
function mix(a, b) {}
const greeting = "Hello from Fling!";
console.log(greeting);
```

## Language Features

✅ Statically typed with type inference

✅ Physical unit type system (`grams`, `ml`, `cups`, `celsius`, `minutes`, etc.)

✅ Unit compatibility checking at compile time

✅ Recipes: typed functions with parameters and return types

✅ Prep types: structs with named fields

✅ Immutability by default, with `stirring` for mutable variables

✅ Collections with homogeneous typing

✅ Control flow: `taste` for if, `simmer` for while, `batch` for for-each

✅ Loop control: `toss` for break, `skip` for continue

✅ Constant folding and dead code elimination optimizer

✅ Compiles to readable JavaScript

### `bread.fling` - Full recipe with prep struct

```fling
-- A complete bread workflow using a Dough prep, unit literals, and recipe returns.
prep Dough:
  flour: grams
  water: ml
  salt: grams
  yeast: grams

-- Build a hydrated dough from measured flour and water.
recipe make_dough(flour_grams: grams, water_ml: ml) yields Dough:
  ingredient salt_pinches: grams = 10g
  ingredient yeast_scoop: grams = 7g
  serve Dough { flour: flour_grams, water: water_ml, salt: salt_pinches, yeast: yeast_scoop }

-- The bake recipe accepts oven temperature and bake time as checked units.
recipe bake(oven_heat: celsius, bake_time: minutes) yields label:
  step note_heat: display(oven_heat)
  step note_time: display(bake_time)
  serve "Crusty country loaf"

ingredient loaf: Dough = make_dough(500g, 325ml)
ingredient oven: celsius = 220celsius
ingredient minutes_in_oven: minutes = 35minutes
ingredient bread_label: label = bake(oven, minutes_in_oven)

step show_water: display(loaf.water)
step show_result: display(bread_label)
serve loaf.flour
```

### `smoothie.fling` - Collections and batch loops

```fling
-- Smoothie batching with typed units, a serving scale, and a label collection.
ingredient base_banana: count = 2
ingredient base_yogurt: grams = 150g
ingredient base_milk: ml = 240ml
ingredient base_honey: tbsp = 1tbsp
ingredient ingredient_names: [label] = ["banana", "yogurt", "milk", "honey"]

recipe scale_smoothie(servings: count) yields nothing:
  ingredient bananas: count = base_banana * servings
  ingredient yogurt: grams = base_yogurt * servings
  ingredient milk: ml = base_milk * servings
  ingredient honey: tbsp = base_honey * servings
  batch name in ingredient_names:
    step show_name: display(name)
  step show_bananas: display(bananas)
  step show_yogurt: display(yogurt)
  step show_milk: display(milk)
  step show_honey: display(honey)

step make_three_smoothies: scale_smoothie(3)
serve "Smoothie batch ready"
```

### `countdown.fling` - Simmer loop with toss

```fling
-- Countdown timer using a mutable stirring ingredient and simmer loop.
stirring ingredient count: count = 5

simmer count > 0:
  step show_count: display(count)
  step lower_count: count = count - 1
  taste count == 0:
    toss

serve "Countdown complete"
```

### `unit_converter.fling` - Unit type system showcase

```fling
-- Unit checking keeps weight math with weight and volume math with volume.
ingredient flour_bag: kg = 1kg
ingredient flour_scoop: grams = 250g
ingredient batter_cups: cups = 2cups
ingredient extra_milk: ml = 120ml

-- Same-dimension arithmetic is accepted and normalized internally.
ingredient total_flour: grams = flour_bag + flour_scoop
ingredient total_liquid: ml = batter_cups + extra_milk

-- convert returns the requested compatible unit for serving or display.
ingredient flour_in_kg: kg = convert(total_flour, "kg")
ingredient liquid_in_cups: cups = convert(total_liquid, "cups")

step show_total_flour: display(total_flour)
step show_total_liquid: display(total_liquid)
step show_liquid_cups: display(liquid_in_cups)
serve flour_in_kg
```

## The Unit Type System

Fling treats physical units as first-class static types. Unit literals are normalized internally for code generation, but the analyzer preserves their dimensions so invalid combinations are rejected before JavaScript is emitted. Weight can combine with weight, volume with volume, time with time, and temperature with temperature. Cross-dimension arithmetic, like adding flour to milk, is a compile-time `TypeError`.

| Dimension | Supported units |
| --- | --- |
| Weight | `grams`, `kg`, `oz`, `lbs` |
| Volume | `ml`, `liters`, `cups`, `tbsp`, `tsp`, `floz` |
| Temperature | `celsius`, `fahrenheit` |
| Time | `seconds`, `minutes`, `hours` |

| Operation | Result |
| --- | --- |
| `200g + 100g` | valid, `300g` |
| `1cups + 236.6ml` | valid, `473.2ml` after normalization |
| `500g * 2` | valid, `1000g` |
| `500g / 500g` | valid, `1.0` as `amount` |
| `200g + 350ml` | invalid, `TypeError` |

## Static Checks

Fling performs 36 static checks across scope, type safety, control flow, access, and recipe calls.

### Scope

- Use before declaration
- Duplicate variable in same scope
- Duplicate recipe name
- Duplicate prep name
- Duplicate field in prep
- Undeclared identifier

### Types

- Unit dimension mismatch in arithmetic
- Unit multiplication rules
- Incompatible unit division
- Modulo on unit types
- Comparing incompatible dimensions
- Equality between different types
- `and`/`or` on non-`truth` values
- `not` on non-`truth` value
- Negation of non-numeric
- Condition must be `truth` type for `taste` and `simmer`
- Mixed type collection elements
- Index access on non-collection
- Non-`count` index type
- Field access on non-struct
- Nonexistent field
- Unknown prep type in struct literal
- Non-prep identifier used as a prep type
- Missing field in struct literal
- Extra field in struct literal
- Wrong field type in struct literal
- Explicit type annotation mismatch
- Type inference consistency

### Control Flow

- `toss` outside loop
- `skip` outside loop
- Missing `serve` on all paths
- `serve` type mismatch
- `serve` with expression in `yields nothing` recipe

### Access

- Assignment to immutable ingredient

### Recipe Calls

- Wrong argument count
- Wrong argument type

## Optimizations

- Constant folding: `5 + 3` becomes `8` at compile time
- Unit normalization: `1cups + 236.6ml` becomes `473.2ml`
- Dead code elimination: unreachable statements after `serve` are removed
- Strength reduction: `x * 1` becomes `x`, and `x + 0` becomes `x`

// Generated by the Fling compiler
// Do not edit by hand
function display(x) { console.log(x); }
function size(col) { return col.length; }
const __unitFactors = { grams: 1, kg: 1000, oz: 28.35, lbs: 453.59, ml: 1, liters: 1000, cups: 236.6, tbsp: 14.79, tsp: 4.93, floz: 29.57, celsius: 1, seconds: 1, minutes: 60, hours: 3600 };
function convert(val, unit) {
  if (unit === "fahrenheit") return val * 9 / 5 + 32;
  return val / __unitFactors[unit];
}
function mix(a, b) {}
function make_dough(flour_grams, water_ml) {
  const salt_pinches = 10;
  const yeast_scoop = 7;
  return ({ flour: flour_grams, water: water_ml, salt: salt_pinches, yeast: yeast_scoop });
