Fling

A recipe-themed language where your units actually mean something

View on GitHub Get Started

The Story

Programming and cooking share the same DNA. You gather ingredients, follow a process, and yield something worth serving. Fling takes that metaphor seriously — programs are collections of recipes, variables are ingredients, and every function serves a result. The syntax reads like a recipe card, which means your code tells a story instead of just executing instructions.

Fling's killer feature is a compile-time unit type system. Values like 200g, 350ml, 2cups, and 35minutes aren't just numbers — they carry physical meaning the compiler understands. Add grams to grams and Fling is delighted. Add grams to milliliters and the compiler sends it straight back to the kitchen with a clear error before a single line of JavaScript is emitted.

Under the hood, Fling is a complete compiler pipeline: an Ohm.js grammar, a scope-aware semantic analyzer with 36 static checks, an AST optimizer with constant folding and dead code elimination, and a JavaScript code generator. It was built as a compiler design project at Loyola Marymount University, and compiles to clean, readable JavaScript you would actually want to read.

Fling in Action

Five complete programs covering every syntactic form

Example 1 — Hello World

The simplest Fling program. An ingredient, a serve.

Fling 
ingredient greeting: label = "Hello from Fling!"
serve greeting
JavaScript 
const greeting = "Hello from Fling!";
console.log(greeting);

Example 2 — Unit Types

Unit literals carry physical meaning. The compiler checks every operation for dimensional compatibility.

Fling 
ingredient flour: grams = 200g
ingredient extra: grams = 100g
ingredient total: grams = flour + extra
serve total
JavaScript 
const flour = 200;
const extra = 100;
const total = (flour + extra);
console.log(total);

Example 3 — Recipes and Prep Types

Recipes are typed functions. Prep types are structs. Every field is checked at compile time.

Fling 
prep Dough:
  flour: grams
  water: ml
  salt: grams
  yeast: grams

recipe make_dough(flour_grams: grams, water_ml: ml) yields Dough:
  ingredient salt_pinches: grams = 10g
  ingredient yeast_scoop: grams = 7g
  serve Dough { flour: flour_grams, water: water_ml, salt: salt_pinches, yeast: yeast_scoop }

ingredient loaf = make_dough(500g, 325ml)
serve loaf.flour
JavaScript 
function make_dough(flour_grams, water_ml) {
  const salt_pinches = 10;
  const yeast_scoop = 7;
  return ({ flour: flour_grams, water: water_ml, 
            salt: salt_pinches, yeast: yeast_scoop });
}
const loaf = make_dough(500, 325);
console.log(loaf.flour);

Example 4 — Loops and Collections

Batch loops iterate typed collections. Simmer loops run while a truth condition holds.

Fling 
ingredient items: [label] = ["flour", "water", "salt", "yeast"]

batch item in items:
  step show: display(item)

stirring ingredient count: count = 5
simmer count > 0:
  step show_count: display(count)
  step decrement: count = count - 1
  taste count == 0:
    toss

serve "Done"
JavaScript 
const items = ["flour", "water", "salt", "yeast"];
for (const item of items) {
  display(item);
}
let count = 5;
while ((count > 0)) {
  display(count);
  count = (count - 1);
  if ((count === 0)) { break; }
}
console.log("Done");

Example 5 — Unit Converter

Same-dimension units combine freely. The optimizer normalizes mixed units at compile time.

Fling 
ingredient flour_bag: kg = 1kg
ingredient flour_scoop: grams = 250g
ingredient batter_cups: cups = 2cups
ingredient extra_milk: ml = 120ml

ingredient total_flour: grams = flour_bag + flour_scoop
ingredient total_liquid: ml = batter_cups + extra_milk

step show_flour: display(total_flour)
step show_liquid: display(total_liquid)
serve total_flour
JavaScript 
const flour_bag = 1000;
const flour_scoop = 250;
const batter_cups = 473.2;
const extra_milk = 120;
const total_flour = (flour_bag + flour_scoop);
const total_liquid = (batter_cups + extra_milk);
display(total_flour);
display(total_liquid);
console.log(total_flour);

The Unit Type System

Fling treats physical units as compile-time types. The analyzer tracks dimensions through every expression and rejects cross-dimension operations before code generation begins.

Dimension Units
Weight grams, kg, oz, lbs
Volume ml, liters, cups, tbsp, tsp, floz
Temperature celsius, fahrenheit
Time seconds, minutes, hours

✅ Valid

200g + 100g → 300g 1cups + 236.6ml → 473.2ml 500g * 2 → 1000g 500g / 500g → 1.0 (amount)

❌ Invalid

200g + 350ml → TypeError 35minutes + 100celsius → TypeError 200g * 350ml → TypeError

36 Static Checks at Compile Time

Scope (6 checks)

  • Use before declaration
  • Duplicate variable in same scope
  • Duplicate recipe name
  • Duplicate prep name
  • Duplicate field in prep
  • Undeclared identifier

Types (18 checks)

  • Unit dimension mismatch in arithmetic
  • Unit multiplication rules (unit × unit is invalid)
  • Incompatible unit division
  • Modulo on unit types
  • Comparing incompatible dimensions
  • Equality between different types
  • and/or on non-truth values
  • not on non-truth value
  • Negation of non-numeric
  • Condition must be truth type
  • Mixed type collection elements
  • Index access on non-collection
  • Non-count index type
  • Field access on non-struct
  • Nonexistent field
  • Unknown prep type in struct literal
  • Missing or extra fields in struct literal
  • Explicit type annotation mismatch

Control Flow (5 checks)

  • toss outside loop
  • skip outside loop
  • Missing serve on all paths
  • serve type mismatch
  • serve with expression in yields nothing recipe

Access (1 check)

  • Assignment to immutable ingredient

Recipe Calls (2 checks)

  • Wrong argument count
  • Wrong argument type

Get Started

Clone and install 
git clone https://github.com/FewBlue04/Fling
cd Fling
npm install
npm test
Compile a program 
node src/fling.js yourprogram.fling
CLI flags 
node src/fling.js --syntax file.fling    # syntax check only
node src/fling.js --parse file.fling     # print AST as JSON
node src/fling.js --analyze file.fling   # type check only
node src/fling.js --optimize file.fling  # print optimized AST
node src/fling.js --chef                 # 🍳 surprise
